## Communication
- 모든 설명은 한국어로 한다.
- 작업 시작 전, 무엇을 할지 1~2문장으로 먼저 설명한다.
- 파일을 수정하기 전, 어떤 파일을 왜 수정하는지 먼저 설명한다.
- 작업 완료 후 아래 형식으로 정리한다:
  1. 무엇을 바꿨는지
  2. 왜 바꿨는지
  3. 검증은 어떻게 했는지
- 가능하면 파일 경로를 함께 적는다.
- 설명은 초보자도 이해할 수 있게 쉽게 쓴다.
- 너무 길게 쓰지 말고 핵심만 정리한다.
- 작업이 끝나면 다음 무엇을 해야하는지 설명한다.
- 명령어는 powershell말고 terminal로 실행한다.
- 명령어를 전달해 줄 때도 termianl 이나 cmd 형식으로 전달한다.
- 코드를 생성할 때 주석을 생성하는데 보통은 함수 단위로 하지만, 중요한 로직이면 주석을 꼭 달아서 설명한다.
